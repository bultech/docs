# Daily Report Details and Data Entry

When you select a day from the [calendar view](/en/daily-reports/report-view), a detail window opens containing all reporting categories for that day.

## Data Entry and Editing Steps

<Steps>
  <Step title="Select a Category">
    Select the category you want to enter data for (e.g., Labor, Equipment, Photos) from the left menu of the window.
  </Step>
  <Step title="Add a New Row">
    Start data entry by clicking on the empty row at the bottom of the table in the content area.
  </Step>
  <Step title="Enter the Data">
    Enter the values by clicking on the relevant cells (e.g., Number of Workers, Hours Worked). All data you enter is saved automatically.
  </Step>
</Steps>

## Interface and Other Actions

<AccordionGroup>
  <Accordion title="Top Area Actions">
    - **Date Navigation:** Allows you to quickly switch between days.
    - **Submit & Publish Button:** Sends the reports for that day via email to the distribution list specified in the settings.
    - **Saved Status:** An indicator showing that your changes have been automatically saved.
  </Accordion>
  <Accordion title="Adding Photos">
    When you select the **Photos** category, a file upload area appears. You can upload site photos for that day by clicking the "**Add Photo**" box or by using drag-and-drop.
  </Accordion>
  <Accordion title="Deleting a Row">
    To delete a row, hover over it with the mouse, right-click, and select "Delete Row".
  </Accordion>
</AccordionGroup>

## Important Notes

<AccordionGroup>
    <Accordion title="Authorization">
        Data entry, editing, and deletion actions are subject to the user's [roles and permissions](/en/platform-rules/roles-and-permissions-rbac).
    </Accordion>
    <Accordion title="Report Approval Process">
        Entered reports are generally subject to an approval process. Authorized users, such as a Project Manager, can review reports and mark them as "Approved" or "Rejected." These statuses are indicated by color codes in the main calendar view.
    </Accordion>
</AccordionGroup>
